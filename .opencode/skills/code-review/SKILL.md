# Skill: Code Review

When asked to review code, follow these guidelines:

## Review Checklist
- **Security**: Check for injection vulnerabilities, authentication issues, and data exposure
- **Performance**: Look for N+1 queries, memory leaks, and unnecessary computations
- **Best Practices**: Follow language-specific conventions, DRY principles, and proper error handling
- **Testing**: Verify test coverage includes edge cases and error states
- **Documentation**: Check if public APIs and complex logic are documented

## Output Format
Always structure reviews as:
1. **Summary** — Brief overview of the code quality
2. **Critical Issues** — Bugs or security concerns
3. **Suggestions** — Improvements and optimizations
4. **Positive Notes** — What was done well
